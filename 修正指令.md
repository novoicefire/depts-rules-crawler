二、修改 scripts/fetch_requirements.py：改成 thread-local SessionPool，不要每 task new session

目標：

1. 不要讓多個 thread 共用同一個 requests.Session。
2. 也不要每個 task 都建立新的 Session，避免對學校系統建立太多連線。
3. 每個 worker thread 在同一個年度同一個 retry round 內重用自己的 Session。
4. 每個年度開始時獨立。
5. 每個 retry round 開始時重置 session pool，讓 invalid retry 換新 session。

新增 import：

import threading

新增 class：

class SessionPool:
"""
requests.Session is not thread-safe.
Use one session per worker thread to reuse connections without sharing mutable session state across threads.
A new SessionPool should be created for each retry round.
"""
def **init**(self):
self.local = threading.local()

```
def get_session(self):
    if not hasattr(self.local, "session"):
        self.local.session = create_session()
    return self.local.session
```

修改 fetch_with_delay：

def fetch_with_delay(year, deptid, class_code, timeout, delay, session_pool):
if delay and delay > 0:
time.sleep(delay)
session = session_pool.get_session()

3. 每個 retry round 建立新的 SessionPool：
